Budget changes do not trigger learning periods in Google Ads or Microsoft Ads. The algorithm learns from conversion signals and auction data — budget is a delivery cap, not a learning input. Challenge the "one change a week" rule when raised by clients or cited as a reason to delay a fix.

**Why:** Google's documented learning period triggers are bid strategy changes, tCPA/tROAS target shifts, pauses, and major targeting changes. Budget is not on that list. The "don't change budgets" advice is commonly repeated by Google reps but is not grounded in how the algorithm works — user's view is that Google reps are outsourced and poorly trained, passing on outdated received wisdom rather than accurate guidance.

**Where the concern has merit:** Constant yo-yoing (raising and lowering every few days) creates inconsistent delivery that makes data harder to read — a measurement problem, not an algorithmic one. Aggressively cutting budget and going dark can make re-ramping rough.

**How to apply:** A one-time correction (e.g., fixing a misconfigured daily cap) is not a harmful change — make it cleanly and leave it alone. Batch any budget changes so a campaign gets one deliberate move, not multiple adjustments. Don't delay legitimate fixes because of the "one change a week" rule. If a client raises this objection, address it directly with the distinction between budget (delivery cap) and bid strategy (what the algorithm actually learns from).
