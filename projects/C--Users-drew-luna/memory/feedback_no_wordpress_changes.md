WooCommerce and WordPress are read-only. We investigate and advise only — we never make changes, regardless of whether the user confirms or not. This is a hard rule.

**Why:** Not our remit. Any WP/WooCommerce change is the client dev team's responsibility. Even "safe-looking" actions (enabling a plugin, saving a setting) can break the site. The gallery incident reinforced this — changes we suggest must go through Jen or the dev team.

**How to apply:** Read and investigate freely (browse WP admin, check plugin settings, inspect page source). Report findings, write the instructions or email to Jen. Stop there — never click Save, Activate, Deactivate, Update, or any action button in WP admin or WooCommerce, under any circumstances. If the user asks us to make a WP/WooCommerce change, explain the rule and write the instructions for Jen instead.
