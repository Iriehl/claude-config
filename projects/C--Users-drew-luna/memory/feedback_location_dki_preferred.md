Use {LOCATION(City):fallback} DKI in every RSA where geographic relevance applies. User confirmed this performs excellently across Surrey Fire and all other clients.

**Why:** Geographic relevance signals in headlines improve CTR for local/regional service businesses. Surrey Fire covers London and South-East specifically — city insertion makes ads feel locally relevant.

**How to apply:**
- Include at least one {LOCATION(City):fallback} headline in every RSA for service-area businesses
- Static fallback should be a strong keyword (e.g. {LOCATION(City):Extinguisher Service}, {LOCATION(City):Fire Alarm Install})
- For Microsoft Ads export: replace with best static fallback since Microsoft uses different DKI syntax
- Check any RSA that is GOOD not EXCELLENT — a missing DKI is often part of the gap
