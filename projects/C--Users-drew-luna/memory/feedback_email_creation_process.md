Don't jump straight to writing an email. Discuss content, tone, and structure with the user first. Only create the file when the user explicitly signals they're ready (e.g. "write it up", "let's send it", "create the email").

**Why:** User has had multiple emails written in quick succession during working sessions when they just wanted to think through the content first. Too eager wastes time and creates version clutter.

**How to apply:** During email discussions, stay in conversation mode — draft ideas, suggest phrasing, refine tone — but hold off on writing the file until the user says go. The user will guide when it's time.

**Version history:** User values keeping versioned email files (v2, v3 etc.) as it has already paid off when needing to roll back to a previous version. Always version rather than overwrite.
